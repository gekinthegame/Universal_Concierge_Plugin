🔌 Connect a publishing account
Tokens stay on this device only (0600 file) and are used solely to deploy to the host you choose — never shared, never sent anywhere but that platform's own API. IPFS needs no token; these Web2 hosts are optional opt-ins.
✦ New project
Name it, then pick what you're building — the Concierge stages the right starter files. A mobile app/game is set up as an installable PWA from the start.
⬆ Commit project to GitHub
Stages every file in the open project (git add -A, honouring .gitignore — the Concierge seeds one if your project has none), commits it, and pushes to a GitHub repo (created automatically if it doesn't exist). Uses your connected GitHub token. Pushing source is egress — password-gated.
📌 Connect a pinning service
Pinning keeps your IPFS site online when this node is off — the service hosts the content on its always-on nodes. Tokens stay on this device only (0600 file) and go only to that service's API. All four speak the standard IPFS Pinning Services API.
🔥 Kept hot on your node
These records are encrypted and pinned on your private Kubo node, served to your paired devices over your private swarm while this node is up. Unpinning stops serving + frees them; the original record stays in your store.

Fellow Concierges appear as brains; every other network node is a small white star. Positions are approximate (by network region, not exact geo-IP).
